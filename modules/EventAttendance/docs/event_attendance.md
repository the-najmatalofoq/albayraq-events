# EventAttendance Module

## Module Purpose

The EventAttendance module tracks worker check-in and check-out for each day of an event. Attendance is recorded per participation (worker per event) per date. Workers can check in using barcode scanning (generated by supervisors) or admin override. The module enforces geofencing — check-in only allowed within event's geofence radius. Attendance statuses: present, absent, late, excused. This module is critical for payroll calculation and worker performance tracking.

## Table Schema

### `event_attendance_records`

| Column                 | Type          | Constraints                                           | Description                                   |
| ---------------------- | ------------- | ----------------------------------------------------- | --------------------------------------------- |
| id                     | uuid          | PRIMARY KEY                                           | Auto-generated UUID                           |
| event_participation_id | uuid          | FOREIGN KEY → event_participations.id, CASCADE DELETE | Worker's participation                        |
| date                   | date          | NOT NULL                                              | Attendance date (YYYY-MM-DD)                  |
| status                 | string        | NOT NULL                                              | present, absent, late, excused                |
| check_in_at            | timestamp     | NULLABLE                                              | When worker checked in                        |
| check_out_at           | timestamp     | NULLABLE                                              | When worker checked out                       |
| check_in_latitude      | decimal(10,7) | NULLABLE                                              | GPS latitude at check-in                      |
| check_in_longitude     | decimal(10,7) | NULLABLE                                              | GPS longitude at check-in                     |
| check_out_latitude     | decimal(10,7) | NULLABLE                                              | GPS latitude at check-out                     |
| check_out_longitude    | decimal(10,7) | NULLABLE                                              | GPS longitude at check-out                    |
| method                 | string        | NOT NULL                                              | barcode, admin_override                       |
| verified_by            | uuid          | FOREIGN KEY → users.id, NULLABLE                      | Admin who overrode (if method=admin_override) |
| created_at             | timestamp     | NOT NULL                                              |                                               |
| updated_at             | timestamp     | NOT NULL                                              |                                               |

**Unique Constraint:** `(event_participation_id, date)` — one record per worker per day.

**Status Values:**

- `present` — Worker checked in and out (or at least checked in)
- `absent` — Worker did not check in
- `late` — Worker checked in after daily_start_time
- `excused` — Worker absent with valid excuse

## Domain Entities

### Aggregate Root: `AttendanceRecord`

**Identity:** AttendanceId (UUID)

**Core Attributes:**

- **ParticipationId:** Reference to worker's participation
- **Date:** Date of attendance
- **Status:** AttendanceStatus enum
- **CheckInAt:** Timestamp (nullable)
- **CheckOutAt:** Timestamp (nullable)
- **CheckInCoordinates:** GeoCoordinates (nullable)
- **CheckOutCoordinates:** GeoCoordinates (nullable)
- **Method:** AttendanceMethod enum (barcode, admin_override)
- **VerifiedBy:** UserId (nullable, for overrides)

**Business Rules:**

- One record per (participation, date)
- Check-in requires accepted contract
- Check-in requires valid (non-expired) barcode or admin override
- Check-in location must be within event geofence (unless admin override)
- Check-out requires prior check-in
- Cannot check in after event daily_end_time (marked as late)
- Cannot check in on non-event date

### Value Objects

- **AttendanceId:** UUID wrapper
- **AttendanceStatusEnum:** PRESENT, ABSENT, LATE, EXCUSED
- **AttendanceMethodEnum:** BARCODE, ADMIN_OVERRIDE
- **GeoCoordinates:** Latitude, longitude — validates ranges

### Repository Interface: `AttendanceRepositoryInterface`

- `save(AttendanceRecord $record): void`
- `findById(AttendanceId $id): ?AttendanceRecord`
- `findByParticipationAndDate(ParticipationId $participationId, Date $date): ?AttendanceRecord`
- `findByParticipation(ParticipationId $participationId): array`
- `findByEvent(EventId $eventId): array`
- `findAbsentByParticipation(ParticipationId $participationId): array` (dates with no check-in)
- `countPresentDays(ParticipationId $participationId): int`

### Barcode Repository Interface: `BarcodeRepositoryInterface`

- `save(Barcode $barcode): void`
- `findByText(string $barcodeText): ?Barcode`
- `findValidByEvent(EventId $eventId): array` (not expired)
- `delete(BarcodeId $id): void`

### Domain Events

- `WorkerCheckedIn` — When check-in recorded (payload: attendance_id, participation_id, date, time, location)
- `WorkerCheckedOut` — When check-out recorded
- `AttendanceMarkedLate` — When check-in after daily_start_time
- `AttendanceMarkedAbsent` — When end of day with no check-in (batch job)
- `BarcodeGenerated` — When supervisor creates barcode

## Infrastructure Implementation

### Eloquent Models

**AttendanceRecordModel:**

- Table: `event_attendance_records`
- Casts: `date` → date, `check_in_at` → datetime, `check_out_at` → datetime
- Relationships: `participation()`, `verifier()`

**AttendanceBarcodeModel:**

- Table: `attendance_barcodes`
- Casts: `expires_at` → datetime
- Relationships: `event()`, `supervisor()`

### EloquentAttendanceRepository

Implements AttendanceRepositoryInterface.

**Key methods:**

- `save()` → creates or updates record (upsert by participation_id + date)
- `findByParticipationAndDate()` → unique record lookup
- `findAbsentByParticipation()` → dates in event range with no check-in
- `countPresentDays()` → for payroll calculation

### Geofence Validation Service

- Checks if coordinates are within event's geofence radius
- Uses Haversine formula for distance calculation
- Returns boolean + distance in meters

## Security and Validation Rules

### Authorization Rules

| Action                | Required Role                                                |
| --------------------- | ------------------------------------------------------------ |
| Check-in/out          | Worker (own participation only)                              |
| Override attendance   | project_manager, area_manager, site_manager                  |
| Generate barcode      | supervisor, project_manager                                  |
| View event attendance | project_manager, area_manager, site_manager, general_manager |
| View own attendance   | Worker (self)                                                |

### Validation Rules

**CheckIn:**

- `participation_id`: required, exists, participation must be active
- `barcode_text`: required, exists, not expired
- `latitude/longitude`: required, must be within event geofence
- Contract must be accepted
- Event must be active (status = ACTIVE)

**CheckOut:**

- Must have check-in record for the day
- Check-out time must be after check-in time

**OverrideAttendance:**

- `status`: required, in: present, absent, excused
- `verified_by`: required, must be authorized role

## Performance Considerations

- **Indexes:** `(event_participation_id, date)` unique, `status`, `date`
- **Composite index:** `(event_participation_id, status)` for summary queries
- **Batch job:** Mark absent runs at event daily_end_time, processes all participations without check-in
- **Geofence caching:** Event coordinates cached for 1 hour to reduce DB hits
- **Barcode lookup:** Index `barcode_text` for fast validation during check-in
